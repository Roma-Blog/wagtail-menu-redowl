# Wagtail Menu от RedOwl

Гибкое древовидное меню для Wagtail CMS. Аналог системы меню WordPress, реализованный для Wagtail.

## Описание

Wagtail Menu — это приложение для управления навигационными меню в проектах на Wagtail. Оно предоставляет интуитивно понятный интерфейс для создания, редактирования и организации пунктов меню с поддержкой вложенности до 4 уровней.

### Возможности

- Древовидная структура меню
- Три типа ссылок: страницы Wagtail, внешние URL, якорные ссылки
- AJAX-перемещение пунктов меню без перезагрузки страницы
- Интеграция в административную панель Wagtail
- Готовые шаблоны и CSS-стили

### Типы ссылок

1. **Страница сайта** — связь с любой страницей в дереве Wagtail
2. **Внешний URL** — произвольный внешний адрес (например, https://example.com)
3. **Якорная ссылка** — ссылка на якорь внутри страницы (например, #contacts)

## Требования

- Python >= 3.10
- Django >= 6.0
- Wagtail >= 7.3

## Быстрый старт

### 1. Создание меню

1. Войдите в административную панель Wagtail (`/admin/`)
2. В главном меню выберите пункт **Menus**
3. Нажмите **Add Menu**
4. Укажите название (например, "Главное меню") и слаг (например, `main_menu`)
5. Сохраните

### 2. Добавление пунктов меню

1. На странице редактирования меню нажмите **Add Item**
2. Заполните поля:
   - **Тип ссылки** — выберите тип (страница, внешний URL, якорь)
   - **Страница** — выберите страницу Wagtail (для типа "Страница сайта")
   - **URL** — укажите внешний адрес или якорь (для типов "Внешний URL" и "Якорная ссылка")
   - **Заголовок** — текст пункта меню (если не заполнен, используется заголовок страницы)
   - **Порядок** — число для сортировки пунктов
   - **Открывать в новой вкладке** — для внешних ссылок
3. Сохраните пункт

### 3. Создание подменю

Для создания вложенного меню:

1. Создайте родительский пункт
2. Создайте дочерний пункт
3. В поле **Родительский пункт** выберите созданный родительский пункт
4. Сохраните

Повторяйте для создания многоуровневой структуры.

### 4. Отображение меню в шаблоне

В базовом шаблоне сайта (обычно `base.html`) добавьте:

```django
{% load menu_tags %}

<header>
    <div class="logo">LOGO</div>
    {% render_menu 'main_menu' %}
</header>
```

Где `main_menu` — слаг созданного вами меню.

## Использование в шаблонах

### Базовый вызов

```django
{% load menu_tags %}
{% render_menu 'main_menu' %}
```

### С кастомным шаблоном

```django
{% load menu_tags %}
{% render_menu 'main_menu' template='menu/custom.html' %}
```

### С ограничением глубины

```django
{% load menu_tags %}
{% render_menu 'main_menu' max_depth=2 %}
```

### Получение объекта меню

```django
{% load menu_tags %}
{% get_menu 'main_menu' as menu %}
{% if menu %}
    <h2>{{ menu.name }}</h2>
{% endif %}
```

## Структура шаблонов

Приложение предоставляет следующие шаблоны:

```
wagtail_menu/
├── templates/
│   └── wagtail_menu/
│       ├── menu.html           # Базовый шаблон меню
│       └── menu_item.html      # Шаблон пункта меню (рекурсивный)
└── static/
    └── wagtail_menu/
        └── css/
            └── menu.css        # Базовые стили меню
```

### Кастомизация шаблонов

Для изменения внешнего вида меню:

1. Скопируйте шаблоны из `wagtail_menu/templates/wagtail_menu/` в ваш проект
2. Внесите необходимые изменения
3. Укажите путь к кастомному шаблону в теге `{% render_menu %}`

## Административная панель

### Управление порядком пунктов

- **Стрелки вверх/вниз** — перемещение пункта на одну позицию
- **Перетаскивание** — drag-and-drop за иконку перемещения
- Порядок сохраняется автоматически через AJAX

### Настройки пункта меню

| Поле | Описание |
|------|----------|
| Тип ссылки | Определяет тип пункта (страница, внешний URL, якорь) |
| Страница | Связь со страницей Wagtail |
| URL | Внешний адрес или якорь |
| Заголовок | Текст пункта (переопределяет заголовок страницы) |
| Родительский пункт | Для создания вложенной структуры |
| Порядок | Число для ручной сортировки |
| Открывать в новой вкладке | Добавляет `target="_blank"` к ссылке |

## Модели данных

### Menu

Основная модель, представляющая контейнер меню.

| Поле | Тип | Описание |
|------|-----|----------|
| name | CharField | Название меню |
| slug | SlugField | Уникальный идентификатор для использования в шаблонах |
| is_active | BooleanField | Флаг активности меню |

### MenuItem

Модель пункта меню.

| Поле | Тип | Описание |
|------|-----|----------|
| menu | ForeignKey | Связь с родительским меню |
| parent | ForeignKey | Ссылка на родительский пункт (для вложенности) |
| link_type | CharField | Тип ссылки (page, custom, anchor) |
| page | ForeignKey | Связь со страницей Wagtail |
| title | CharField | Заголовок пункта |
| url | CharField | Внешний URL или якорь |
| sort_order | IntegerField | Порядок сортировки |
| open_in_new_tab | BooleanField | Открывать в новой вкладке |

## Примеры использования

### Главное меню в хедере

```django
<!-- base.html -->
{% load menu_tags %}

<header class="site-header">
    <div class="container">
        <a href="/" class="logo">Мой сайт</a>
        <nav class="main-navigation">
            {% render_menu 'main_menu' %}
        </nav>
    </div>
</header>
```

### Меню в футере

```django
<!-- base.html -->
{% load menu_tags %}

<footer class="site-footer">
    <div class="container">
        <nav class="footer-navigation">
            {% render_menu 'footer_menu' %}
        </nav>
    </div>
</footer>
```

### Мобильное меню

```django
<!-- base.html -->
{% load menu_tags %}

<nav class="mobile-menu">
    {% render_menu 'mobile_menu' max_depth=3 %}
</nav>
```

## Устранение неполадок

### Меню не отображается

1. Проверьте, что меню с указанным слагом существует
2. Убедитесь, что в меню есть активные пункты
3. Проверьте, что тег `{% load menu_tags %}` добавлен в шаблон

### Пункты не перемещаются

1. Проверьте подключение JavaScript в административной панели
2. Откройте консоль браузера (F12) и проверьте наличие ошибок
3. Убедитесь, что у пользователя есть права на редактирование меню

### Ошибка при миграции

```bash
python manage.py migrate wagtail_menu
```

Если возникает ошибка, убедитесь, что `wagtail_menu` добавлен в `INSTALLED_APPS`.
